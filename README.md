## FILE testing1.mid check
> Header chunk
0x4d, 0x54, 0x68, 0x64, -> MThd aka the header
0x00, 0x00, 0x00, 0x06, -> Lenght of header chunk (6 bytes)
0x00, 0x01, -> format (tday I'll be usin only 1, aka multi track one ;]])
0x00, 0x02, -> the tracks that follow, for this example there is 2
0x00, 0x80, -> the value that is responsibble for some time wizardy

> Track chunk
0x4d, 0x54, 0x72, 0x6b, -> MTrk aka the header for track chunk
0x00, 0x00, 0x00, 0x43, -> the number of bytes for track event (67 next bytes are track event)

>Track event
0x00, 0xff, -> Means that there will be Meta event
0x03, -> Meta event type (this one is "Sequence or track name")
0x10, -> lenght of Meta event in bytes (next 16 bytes)
0x54, 0x72, 0x69, 0x70, \
0x6c, 0x65, 0x4f, 0x73,  \ This in char is "TripleOscillator"
0x63, 0x69, 0x6c, 0x6c,  /
0x61, 0x74, 0x6f, 0x72, /

0x00, 0xff, -> Means that there will be Meta event
0x51, -> Meta event type (this one is "Tempo setting")
0x03, -> lenght of Meta event in bytes (next 3 bytes)
0x06, 0x8a, 0x1b, -> more magic time wizardy (428571 ms)

0x00, 0x90, -> delta 0 the 0x90 goes to bin -> 1001 is setting that note is pressed and 0000 sets on which channel
0x3c, -> key number wich is 60 aka 5C
0x40, -> velocity with what this note is pressed, aka loudnes wich is 64

0x81, 0x00, -> is ectra delay since 0x81 is 1000-0001 and since the first number is 1 means it extends to next one, aka variale size and it specifies delta time, for this value is 256
0x80, -> this releses the note
0x3c, -> key number wich is 60 aka 5C
0x40, -> velocity with what this note is pressed, aka loudnes wich is 64

0x00, 0x90, -> note pressed
0x3d, -> same as previous one but higer octave
0x40, 

0x81, 0x00, 0x80, -> note relesed
0x3d, 
0x40, 

0x00, 0x90, -> note pressed
0x3e, 
0x40,

0x3e,

0x3f,

0x00, 0xff, -> means this is a event

> End of first track chunk
