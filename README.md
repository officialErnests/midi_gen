> Header chunk
0x4d, 0x54, 0x68, 0x64, -> MThd aka the header
0x00, 0x00, 0x00, 0x06, -> Lenght of header chunk (6 bytes)
0x00, 0x01, -> format (tday I'll be usin only 1, aka multi track one ;]])
0x00, 0x02, -> the tracks that follow, for this example there is 2
0x00, 0x80, -> the value that is responsibble for some time wizardy

> Track chunk
0x4d, 0x54, 0x72, 0x6b, -> MTrk aka the header for track chunk
0x00, 0x00, 0x00, 0x43, -> the number of bytes for track event (67 next bytes are track event)

>Track event
0x00, 0xff, -> Means that there will be Meta event
0x03, -> Meta event type (this one is "Sequence or track name")
0x10, -> lenght of Meta event in bytes (next 16 bytes)
0x54, 0x72, 0x69, 0x70, \
0x6c, 0x65, 0x4f, 0x73,  \ This in char is "TripleOscillator"
0x63, 0x69, 0x6c, 0x6c,  /
0x61, 0x74, 0x6f, 0x72, /

0x00, 0xff, -> Means that there will be Meta event
0x51, -> Meta event type (this one is "Tempo setting")
0x03, -> lenght of Meta event in bytes (next 3 bytes)
